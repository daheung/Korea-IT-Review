2) 조건문
	if문, switch문

- if ~ else if ~ else문
	if (조건식1) { 
		조건식1이 true일 때 실행문;
	} else if (조건식2) {
		조건식1이 false이고 조건식2가 true일 때 실행문;
	} else {
		조건식1, 2가 모두 false일 때 실행문;
	}


- if ~ if문 : 무조건 if문을 만나면 모든 조건을 다 확인한다!
	if (조건식){
		//조건식 true일 때 실행문;
	}
	if (조건식){
		//조건식 true일 때 실행문;
	}

- if ~ else if문 : if문의 조건식을 확인하여 true면 else if문을 실행하지 않고 
			false면 else if문을 실행한다

- 중첩 if문
	첫번째 조건이 true 일 때만 그 안에 있는 두번째 if문이 실행된다
	조건이 많아질수록 논리 흐름을 잘 정리하는 것이 중요하며
	중괄호 영역과 들여쓰기를 잘 작성한다

	if(조건1) {
		if (조건2) {
			실행문;
		}
	}

- 중첩 if문을 사용하는 경우
	조건을 단계별로 검사해야할 때
	앞 조건이 거짓이면 뒤 조건을 아예 검사하지 않아야할 때
	로그인 -> 권한(관리자/사용자) -> 세부권한(사용자-일반사용자/판매자) 같은 구조일 때 사용

- 주의사항
	중첩이 깊어질수록 가독성이 나빠짐
	3단계 이상 중첩되면 else if를 쓰거나 메소드를 분리하는 것을 고려하는 것이 좋음
	중괄호 {} 생략불가

1. 반복문
1) 반복문이 필요한 이유
	같은 코드(출력/입력/계산)를 여러번 해야할 때
	사람이 직접 작성하면 실수가 발생할수있고 수정하기가 힘들기 때문에
	=> 반복문은 규칙으로 반복시켜준다

2) 반복문 종류
	for문
	while문
	do ~ while문

2. for문
	반복횟수나 범위가 정해졌을 때

- 문법
	for(초기식; 조건식; 증감식) {
		반복 실행할 문장(실행문);
	}

초기식 : 시작점
조건식 : 언제까지 반복할지
증감식 : 다음 반복으로 갈 수 있도록 값을 변화

- 실행순서 ★
	초기식 1번 실행
	조건식 검사
	조건식의 결과가 true면 {}안의 실행문 실행
	증감식 실행
	다시 조건식 검사 ... 반복
	조건식의 결과가 false가 되는 순간 종료

- 증감식에 i++를 사용하는 이유 i-- i+=2
(1) 가장 단순한 변화 파악 가능
	한 번 반복할 때마다 1씩 증가
	반복 횟수를 셀 때 가장 직관적

(2) 조건식과의 궁합이 좋음
	언젠가 조건식은 무조건 false가 되어야함

(3) 몇 번 반복했는지 바로 알 수 있음
	i 자체가 반복 횟수 역할
	출력, 계산, 배열 인덱스에 활용 가능

for(int i = 0; i < 3; i++){

}

for(int i = 0; i < 3; ++i){

}

int i = 0;
while(i < 3){
	syso(i);
	i++;	//i를 1증가 시키는 역할만
}

3. while문
	조건식이 참인동안 반복한다
	실행중에 조건식이 거짓이 될 수 있어야 무한루프에 빠지지 않는다

- 문법
	while(조건식){
		실행문;
	}

4. do ~ while문
	while문과 동일하지만 최소 한 번은 무조건 실행해야할 때 사용한다

- 문법
	do {
		실행문;
	} while(조건식);

5. 기타제어문
	반복문에서 주로 사용한다
	
1) break : 즉시 해당 중괄호 영역을 탈출한다
	- if문은 적용되지 않는다(if문 안에서 사용 시 if문을 감싸고 있는 중괄호 영역을 탈출한다)
	- 아래 코드를 실행시키지 않는다

2) continue : 즉시 다음 반복으로 넘어간다(현재 반복 스킵)
	- 아래 코드를 실행시키지 않는다

[for문 공통 로직 사고법]
	for문 범위가 보이는 반복!!
	범위 -> 처리 -> 필터 -> 종료 순으로 생각
	반복범위를 먼저 정하고 반복 변수의 역할 정의하고 출력/누적 -> 필터 -> 종료 조건 순으로 로직 구성

1) 어디서부터 어디까지 반복하는가? (범위)
	초기식/조건식
	=> 시작값은? 끝값은? 증가인지/감소인지?
2) 반복 변수(i)는 무엇을 의미하는가? 
	i의 역할
	i는 숫자 그 자체인지 / i는 순서(인덱스)인지 / i는 현재 검사중인 값인지?
3) 어떤 값을 출력/누적하는가?
	출력인지 / 합계 누적인지 / 개수 세기인지
4) 어떤 경우를 건너뛸 것인가?
	continue의 대상 => 출력하면 안되는 조건 / 계산하면 안되는 조건
5) 언제 멈출 것인가?
	break의 조건 => 숫자가 특정값에 도달했을때 / 누적결과가 기준을 넘었을 때 / 출력 횟수가 기준에 도달했을 때
6) 종료 기준은 무엇인가? (제일 중요함!!!)
	종료 조건이 i인지 / 결과인지	=> 이거에 정확한 답이 안나오면 로직 전체가 흔들리게 됨

[while문 공통 로직 사고법]
1) 무엇을 기준으로 반복하는가?
	for문과 다르기 때문에 무엇이 반복을 움직이게 하는 변수인지를 먼저 정해야함
2) 무엇을 출력/누적하는가?
	매 반복마다 출력인지/누적 합인지/카운트인지 
3) 무엇을 건너뛸 것인가? => continue
	이번 반복을 무효 처리하고 다음 반복으로 넘어가기
	주의사항 : while문은 증감식이 코드 안에 있기 때문에 continue로 건너뛰면 증감이 안될 수 있음
		continue가 있는 while문은 증감식을 continue전에 두거나 continue 전에 반드시 i를 변화시켜야함
4) 언제 멈출 것인가? => break
	반복 즉시 종료
	break 조건 => 출력 개수 / 합이 기준을 넘었을 때 / 특정값 도달했을때
	주의사항 : break도 위치가 중요함
		특히 포함하고 종료하는 문제의 경우 검사 위치가 핵심!
5) 종료 기준은 반복 변수인가? 출력 결과인가?
	반복변수(i)가 종료 기준인지 
	결과가 종료기준인지
